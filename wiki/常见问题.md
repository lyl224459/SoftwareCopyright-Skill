# 常见问题

### Q: 代码材料是从我的项目里复制的吗？

是的。所有代码材料来自你的真实项目源码。skill 不会用 AI 生成任何代码行。

### Q: 生成的文件在哪里？

在当前项目的 `软件著作权申请资料/正式资料/` 目录下。

### Q: 需要安装什么？

Python 3.10+ 和 python-docx（`pip install python-docx`）。.NET SDK 是可选的，用于完整 DOCX 校验。

### Q: 支持哪些代码助手？

Claude Code（推荐）和 Codex。

### Q: 我的项目不是 Web 前端，能用吗？

可以。skill 支持 Web 后端、桌面应用、CLI 工具、移动端等 9 种项目类型，会自动检测并调整生成文本措辞。

### Q: 代码不足 60 页怎么办？

会自动生成"全部代码材料"，而不是前 30/后 30 页。

### Q: 操作手册能直接用吗？

生成的 DOCX 是本地编辑稿。建议你在 Word/WPS 中检查、修改后，导出为 PDF 再上传官网。

### Q: 申请表信息怎么用？

`申请表信息.txt` 是填报辅助文件。打开中国版权保护中心官网，对照填入对应字段即可。

### Q: 截图怎么处理？

有四种方式：Chrome DevTools MCP 自动截图、Computer Use 桌面截图、自己截图放到指定目录、或跳过截图保留占位符。

### Q: 签章页和合作协议会生成吗？

签章页需要你签名/盖章，合作协议需要各方协商内容，所以 skill 不自动生成。但仓库中提供了参考模板，见 `references/signature_page_template.md` 和 `references/cooperation_agreement_guide.md`。
