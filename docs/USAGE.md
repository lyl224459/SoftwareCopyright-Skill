# 使用指南

## 快速开始

在代码助手中打开需要生成软著资料的项目，直接说：

```text
使用 software-copyright-materials 生成当前项目的软件著作权申请资料
```

Claude Code 也可手动调用：

```text
/software-copyright-materials:software-copyright-materials 读取当前目录中的项目，生成软件著作权申请资料草稿
```

代码助手会按流程引导你填写信息、确认草稿，并在当前项目目录下生成 `软件著作权申请资料/`。

## 工作流程

1. **环境检查** — 自动检查 Python、python-docx、.NET SDK 等依赖
2. **项目定位** — 自动扫描当前目录，找到项目根目录
3. **项目分析** — 分析框架、语言、源码结构
4. **业务理解** — 由模型阅读项目后生成业务理解草稿，需你确认
5. **申请表字段** — 引导你确认软件名称、版本号、著作权人等字段
6. **代码选择** — 生成候选文件清单，你确认抽取哪些源码
7. **Markdown 草稿** — 生成代码材料、申请表信息、操作手册草稿
8. **截图** — 选择截图方式（Codex 浏览器工具 / Chrome DevTools MCP / Computer Use / 自行截图 / 跳过）
9. **确认草稿** — 一次性确认所有 Markdown 草稿
10. **生成正式资料** — 输出 DOCX 和 TXT 到 `软件著作权申请资料/正式资料/`

## 输出文件

```
软件著作权申请资料/
├── 草稿/
│   ├── 业务理解.md / .json
│   ├── 申请表信息.md
│   ├── 代码-前30页.md / 代码-后30页.md
│   ├── 代码文件选择.json
│   ├── 操作手册.md
│   └── 操作手册自检记录.md / .json
├── 正式资料/
│   ├── 申请表信息.txt
│   ├── <软件名称>-代码(前30页).docx
│   ├── <软件名称>-代码(后30页).docx
│   ├── <软件名称>_操作手册.docx
│   └── 生成报告.md
└── 环境检查.md / .json
```

## 关键门禁

以下节点会**停下来等你确认**，确保材料不写偏：

- 完整 DOCX 环境缺失时的处理方式
- 多个候选项目时选哪个
- 业务理解是否准确
- 申请表字段补全
- 代码文件选择
- 截图方式选择
- 最终 Markdown 草稿确认

## 支持的项目类型

本 skill 支持以下项目类型（详见 [PROJECT_TYPES.md](PROJECT_TYPES.md)）：

- Web 前端 / 后端 / 全栈
- 桌面应用（Electron/Tauri/PyQt/WPF 等）
- 移动应用（Flutter/React Native/MAUI 等）
- CLI 命令行工具
- 软件库 / SDK
- 嵌入式软件

支持 30+ 种编程语言和 90+ 个主流框架的自动检测。
